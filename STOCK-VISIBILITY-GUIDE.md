# دليل إدارة حالة المخزون والظهور

## نظرة عامة
تم إضافة نظام متقدم لإدارة حالة المنتجات في المخزون مع ثلاثة خيارات:

### 1. **متوفر (visible)** ✓
- المنتج يظهر بشكل طبيعي في الموقع
- جميع أزرار الشراء متاحة
- يمكن إضافته للسلة والطلب عبر الموقع أو ماسنجر

### 2. **غير متوفر (out_of_stock)** ⚠
- المنتج يظهر في الموقع
- الصور والوصف والأسعار والمقاسات تظهر بشكل طبيعي
- **لكن** أزرار الشراء مخفية
- بدلاً منها تظهر رسالة "غير متوفر حالياً في المخزون"
- يمكن للعملاء رؤية المنتج لكن لا يمكنهم طلبه

### 3. **مخفي (hidden)** ✕
- المنتج لا يظهر نهائياً في:
  - الصفحة الرئيسية
  - صفحة المنتجات
  - نتائج البحث
- إذا حاول أحد فتح رابط المنتج مباشرة، يظهر له "المنتج غير موجود"

---

## كيفية الاستخدام

### من لوحة التحكم (Dashboard)

1. افتح لوحة التحكم: `admin.html`
2. اضغط على زر "المنتجات"
3. في كل منتج، ستجد قائمة منسدلة "حالة المخزون"
4. اختر الحالة المناسبة:
   - **✓ متوفر (يظهر عادي)** - للمنتجات المتاحة
   - **⚠ غير متوفر (يظهر بدون شراء)** - للمنتجات المؤقتة غير المتوفرة
   - **✕ مخفي (لا يظهر نهائياً)** - للمنتجات المتوقفة أو التي لا تريد عرضها

5. التغيير يحدث فوراً ويُسجل في سجل النشاطات

## التحديث المطلوب في قاعدة البيانات

قبل استخدام هذه الميزة، يجب تشغيل هذا الأمر SQL في Supabase:

```sql
-- Add visibility_status column if it doesn't exist
ALTER TABLE product_stock 
ADD COLUMN IF NOT EXISTS visibility_status TEXT DEFAULT 'visible' 
CHECK (visibility_status IN ('visible', 'out_of_stock', 'hidden'));

-- Update existing rows to have 'visible' status if in_stock is true
-- and 'out_of_stock' if in_stock is false
UPDATE product_stock 
SET visibility_status = CASE 
    WHEN in_stock = true THEN 'visible'
    ELSE 'out_of_stock'
END
WHERE visibility_status IS NULL;
```

أو استخدم الملف الجاهز: `STOCK-VISIBILITY-UPDATE.sql`

## الملفات المُحدّثة

### 1. **blocked-setup.sql**
- تم تحديث جدول `product_stock` لإضافة عمود `visibility_status`

### 2. **supabase.js**
- `updateProductStock()` - الآن يدعم `visibilityStatus` كمعامل ثالث
- `getProductStock()` - يُرجع كائن يحتوي على `in_stock` و `visibility_status`
- `fetchProductStock()` - يجلب جميع بيانات المخزون بما فيها `visibility_status`

### 3. **products-data.js**
- `DALAL_PRODUCTS_STOCK` - الآن يحفظ كائن بدلاً من boolean
- `createProductCard()` - يتحقق من `visibility_status` ويخفي المنتجات المخفية

### 4. **product.js**
- `initProductPage()` - يتحقق من `visibility_status`:
  - `hidden` → يعرض صفحة 404
  - `out_of_stock` → يعرض صفحة خاصة بدون أزرار شراء
  - `visible` → يعرض المنتج بشكل طبيعي
- `showOutOfStockProduct()` - دالة جديدة لعرض صفحة المنتج غير المتوفر

### 5. **product.html**
- تم نقل قسم "المقاسات المتاحة" ليكون **تحت الأسعار** بدلاً من فوقها

### 6. **admin.html**
- إضافة قائمة منسدلة لكل منتج لاختيار حالة المخزون
- `loadProductStockStatuses()` - تحميل الحالات الحالية من قاعدة البيانات
- `updateProductStockStatus()` - تحديث حالة المنتج مع تسجيل النشاط

## أمثلة الاستخدام

### مثال 1: منتج نفذ مؤقتاً
```
الحالة: ⚠ غير متوفر (يظهر بدون شراء)
النتيجة: العملاء يشاهدون المنتج ويعرفون أنه موجود لكن غير متوفر حالياً
```

### مثال 2: منتج متوقف نهائياً
```
الحالة: ✕ مخفي (لا يظهر نهائياً)
النتيجة: المنتج يختفي تماماً من الموقع
```

### مثال 3: منتج متوفر
```
الحالة: ✓ متوفر (يظهر عادي)
النتيجة: المنتج يظهر بشكل طبيعي مع جميع أزرار الشراء
```

## ملاحظات مهمة

1. **التغيير فوري**: عند تغيير حالة المنتج من الداشبورد، التغيير يظهر فوراً للعملاء
2. **التسجيل التلقائي**: كل تغيير يُسجل في سجل النشاطات مع التاريخ والوقت
3. **الروابط المباشرة**: حتى لو كان لدى أحد رابط مباشر لمنتج مخفي، لن يستطيع الوصول إليه
4. **SEO**: المنتجات المخفية لا تظهر في نتائج محركات البحث
